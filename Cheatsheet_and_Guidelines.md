# 🚀 Course Success Cheat Sheet & Guidelines

A comprehensive guide balancing academic integrity, systemic problem-solving, development environments, and browser-based tools for modern computer science coursework.

## 🤖 1. AI Acceptable Use Guidelines
Generative AI tools should function as a personalized partner in learning, focusing on conceptual connection rather than automation.

*   **Authorized Deployment:**
    *   Querying for core conceptual explanations (e.g., memory mechanics or structural relationships between objects).
    *   Deconstructing abstract compiler warning logs or syntactic anomalies.
    *   Formulating structural logic and high-level pseudo-code.
*   **Prohibited Deployment:**
    *   Directly reproducing assignment specifications to extract full, functional source code.
    *   Submitting code generated by external tools without explicit authorization.
    *   Employing AI capabilities during exams, locked assessments, or quizzes.
*   **Core Principle:** Students must possess the ability to articulate the underlying mechanics of every line of submitted code.

## 📋 2. AI Programming Assignment Checklist
Execute these validation checkpoints prior to final artifact submission to ensure code quality and structural compliance:

- [ ] **Readability & Standards:** Verify descriptive variable naming and ensure all dead code or temporary debugging outputs are eliminated.
- [ ] **AI Attribution:** Document any AI prompts or collaborative assistance inside the primary file header.
- [ ] **Compilation Compliance:** Confirm the code compiles cleanly under target compiler flags without warnings.
- [ ] **Structural Specifications:** Ensure file nomenclature maps precisely to assignment criteria (e.g., `main.cpp`).
- [ ] **Edge Case Validation:** Rigorously verify boundary conditions, structural constraints, and extreme inputs.

## 🧠 3. Train of Thought Synthesis (TOTS) Learning Model
When navigating complex algorithmic blockages, employ this structured, systematic approach to work hard and resolve logical challenges:

1.  **Deconstruct:** Clearly map out input parameters, required outputs, and absolute design constraints in clear text.
2.  **Branch Alternatives:** Design multiple independent logical trajectories (evaluating structural relationships between objects, matching traditional OOP concepts but focusing on data flowing across distinct entities).
3.  **Evaluate & Backtrack:** Trace sample data points through each path. Explicitly backtrack to previous decision nodes when design criteria or execution boundaries are violated.
4.  **Synthesize:** Translate verified logical thought blocks directly into functional, robust source code.

## 💻 4. Core Development Environments (IDEs)
Select a local development environment matching your operating system and project requirements as suggested by your zyBook:

### Standard Desktop Platforms
*   **Visual Studio Code (VS Code):** Cross-platform, modular ecosystem. Requires configuring a C++ extension alongside native compilers (GCC/Clang).
*   **Microsoft Visual Studio IDE:** Feature-dense enterprise environment native to Windows, optimal for substantial system development.
*   **Xcode:** Optimized native compiler environment engineered specifically for macOS ecosystems.
*   **CLion:** Premium JetBrains environment offering intelligent structural analysis and comprehensive refactoring tools.
*   **Eclipse IDE for C/C++:** Established, open-source platform tailored for structured software engineering.

## 📝 6. Master Markdown (.md) for Your Portfolio
Every software engineering student is strongly encouraged to learn **Markdown (.md)**. It is the industry-standard syntax used to write documentation, project specifications, and professional profiles on platforms like GitHub. Knowing how to present your code beautifully is just as important as writing the code itself.

To quickly learn the layout rules, shorthand syntax, and shortcut symbols, reference the official [Markdown Guide Cheat Sheet](https://www.markdownguide.org/cheat-sheet/).
