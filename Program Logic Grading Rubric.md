As artificial intelligence (AI) becomes an integral part of modern society, integrating it into the evaluation of student learning outcomes is essential for educators.

This 100-point framework evaluates computer science lab assignments by assessing execution success and software engineering practices across four main areas.

> See also **[Programming Assignment Checklist](https://github.com/devservice-ora/AI/blob/main/AI%20Programming%20Assignment%20Checklist.md)**
---

### 1. Program Logic & Functional Correctness (40 Points)

* **Core Requirements Implementation (15 Points):** Program successfully executes all fundamental tasks, business rules, and formulas defined by the prompt specifications.
* **Conditional Logic & Flow (15 Points):** Logical pathways (such as `if`/`else` structures, loops, or switches) evaluate control structures correctly, preventing structural bugs, infinite loops, or logical dead ends.
* **Boundary & Edge-Case Stability (10 Points):** Program handles unexpected inputs, critical thresholds, and extreme data limits flawlessly without breaking or calculating incorrect outputs.

---

### 2. Defensive Programming & Data Validation (20 Points)

* **Input Validation Structures (10 Points):** Explicit error-checking mechanisms (e.g., verifying `std::cin` states) successfully capture out-of-bounds, illegal data types, or logically impossible user inputs before any calculation routines take place.
* **Error Handling & Communication (10 Points):** Provides clear, actionable error messages or warnings to the user, and halts or restarts processing gracefully when data violations occur.

---

### 3. Architecture, Formatting & Maintainability (20 Points)

* **Elimination of Hardcoded Magic Values (10 Points):** Configurable elements, static rates, and system thresholds are abstracted into descriptive, scoped constants, facilitating future modification and clear intent.
* **Console Representation & Formatting (5 Points):** Terminal presentations are clean, consistent, and strictly enforce necessary visual standards (such as uniform alignment, fixed decimal precision, or clean line breaks).
* **Code Styling & Documentation (5 Points):** Follows clean coding conventions, including semantic naming (camelCase or snake_case), uniform indentation, proper bracket placement, and structured comments explaining complex logic blocks.

---

### 4. Final Submission & Analysis (20 Points)

* **Original Deliverable (`Lab-Jim_Pham-Original.cpp`) (5 Points):** An independent, unoptimized, yet fully functional source code file that builds and compiles successfully without syntax errors.
* **AI-Improved Deliverable (`Lab-Jim_Pham-AI-Improved.cpp`) (5 Points):** A refined source code file that explicitly incorporates structural optimizations (e.g., robust input stream checking, constant abstraction, or algorithmic cleanups) and builds cleanly. See also [Course Policy: AI Acceptable Use & The Engineering Mindset](https://github.com/devservice-ora/AI/blob/main/AI%20Acceptable%20Use%20Policy%3A%20Fundamentals%20First.md)
* **Visual Execution Evidence (5 Points):** Clear, legible terminal screenshots are provided, demonstrating successful, bug-free execution runs for both versions of the program.
* **Technical Reflection Paragraph (5 Points):** Includes a concise 3–5 sentence analytical summary detailing the application's underlying mechanics, the evaluation of logical conditions, and the specific structural improvements adopted during the peer/AI optimization process.
